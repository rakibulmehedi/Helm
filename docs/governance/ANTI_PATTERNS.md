# ANTI-PATTERNS

> Behaviors and implementation patterns that are strictly prohibited in the Helm project.

## 1. Product Anti-Patterns
- Feature soup (adding features just to have them)
- Chart overload (too many complex visual analytics)
- Enterprise accounting complexity
- Generic fintech cloning
- AI gimmicks

## 2. Engineering Anti-Patterns
- Giant god widgets (files > 300 lines)
- Random utility dumping
- Deeply nested providers
- Silent architectural rewrites
- Dependency addiction (adding packages instead of writing clean Dart)
- Premature abstractions

## 3. Agentic Anti-Patterns
- Scope expansion without permission
- Implementing future roadmap items prematurely
- Modifying unrelated files
- Ignoring project docs
- Changing product identity silently
- Assuming context instead of reading docs

## 4. UX Anti-Patterns
- Overwhelming dashboards
- Too many colors (violating `AppColors` palette)
- Excessive animations
- Confusing navigation
- Finance jargon overload
